## Линейный поиск

**Линейный поиск** (или последовательный поиск) — это метод поиска элемента в списке (массиве) путём последовательного просмотра каждого элемента списка до тех пор, пока искомый элемент не будет найден или пока все элементы списка не будут проверены.

![Linear Search](https://www.tutorialspoint.com/data_structures_algorithms/images/linear_search.gif)

### Принцип работы

1. Начните с первого элемента списка.
2. Сравните искомый элемент с текущим элементом списка.
3. Если элементы совпадают, верните индекс текущего элемента.
4. Если элементы не совпадают, перейдите к следующему элементу и повторите шаг 2.
5. Если дошли до конца списка и элемент не найден, верните -1 или другое значение, указывающее на то, что элемент не найден.

### Применение

Линейный поиск используется в ситуациях, когда:
- Работа идет с небольшими списками.
- Элементы списка не отсортированы.
- Отсутствует информация о структуре и содержании списка.

### Временная сложность

- Лучший случай: O(1) – когда искомый элемент является первым элементом в списке.
- Худший случай: O(n) – когда элемента нет в списке или он является последним.
- Средний случай: O(n/2) = O(n) – в среднем потребуется проверить половину элементов списка.

### Преимущества и недостатки

**Преимущества:**
- Простота реализации.
- Эффективен для небольших объёмов данных.

**Недостатки:**
- Медленный по сравнению с другими методами поиска, такими как бинарный поиск или поиск с использованием хеш-таблиц.
- Неэффективен для больших объемов данных из-за линейной временной сложности.

### Заключение

Линейный поиск — это базовый и простой метод поиска. Он может быть эффективным для небольших списков, но для больших объемов данных рекомендуется использовать более эффективные алгоритмы.
