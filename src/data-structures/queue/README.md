# Очередь

**Очередь** — это структура данных, представляющая собой коллекцию элементов, поддерживающую две основные операции:

1. **Enqueue** — добавление элемента в конец очереди.
2. **Dequeue** — удаление элемента из начала очереди.

Очередь работает по принципу "первым пришел — первым вышел" (First-In, First-Out, FIFO). Это означает, что первый элемент, добавленный в очередь, также будет первым, который уйдет из нее.

![Очередь](https://upload.wikimedia.org/wikipedia/commons/thumb/5/52/Data_Queue.svg/600px-Data_Queue.svg.png)

## Основные характеристики

- **FIFO (First-In, First-Out)**: Как уже упоминалось, первый элемент, который входит в очередь, также является первым элементом, который выходит из нее.

- **Динамичность**: Размер очереди может изменяться динамически во время выполнения программы.

- **Ограниченное взаимодействие**: Доступ возможен только к первому и последнему элементам очереди. Нет прямого доступа к средним элементам, как это бывает в массивах или списках.

## Основные операции

- **Enqueue**: Добавляет элемент в конец очереди.
- **Dequeue**: Удаляет и возвращает элемент из начала очереди.
- **Peek/Front**: Возвращает элемент из начала очереди без его удаления.
- **IsEmpty**: Проверяет, пуста ли очередь.
- **Size**: Возвращает количество элементов в очереди.

## Временная сложность

|           | Enqueue | Dequeue | Peek | IsEmpty | Size |
|-----------|---------|---------|------|---------|------|
| Сложность | O(1)    | O(1)*   | O(1) | O(1)    | O(1) |

\* Для очереди на основе двух стеков `dequeue` имеет амортизированное время O(1), но может достигать O(n) в худшем случае при перекладывании элементов.

---

## Применение

Очереди часто используются в программировании для решения различных задач, таких как:

- Управление порядком выполнения задач.
- Очереди печати для управления порядком печати документов.
- Очереди сообщений для передачи данных между процессами.
- Алгоритмы обхода в ширину (BFS) в графах.
