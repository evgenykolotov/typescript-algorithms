# Паттерн Singleton

**Паттерн Singleton** — это порождающий паттерн проектирования, который гарантирует, что у класса есть только один экземпляр,
и предоставляет глобальную точку доступа к этому экземпляру. Этот паттерн полезен, когда у вас есть ресурсоемкий объект,
который нужен только один раз в приложении, и вы хотите обеспечить к нему уникальный доступ.

Основные характеристики паттерна Singleton:

* Один единственный экземпляр: Singleton гарантирует, что у класса существует только один экземпляр, который контролируется
и создается самим классом.
* Глобальный доступ: Этот экземпляр класса предоставляет глобальную точку доступа для других частей программы, 
что позволяет управлять ресурсами или данными, к которым нужно иметь единственный доступ.
* Ленивая инициализация: Экземпляр Singleton может создаваться по требованию, а не заранее, что помогает сэкономить ресурсы
и ускорить инициализацию программы.
* Простота использования: Пользовательский код может получить доступ к экземпляру Singleton через статический метод
или свойство, что делает работу с ним удобной и интуитивно понятной.

## Пример на TypeScript

```typescript
class Singleton {
  private static instance: Singleton | null = null; // Приватное статическое поле для хранения экземпляра

  private constructor() {
    // Приватный конструктор, чтобы предотвратить создание экземпляров извне
  }

  public static getInstance(): Singleton {
    // Статический метод для получения экземпляра
    if (Singleton.instance === null) {
      Singleton.instance = new Singleton();
    }
    return Singleton.instance;
  }

  public someMethod(): void {
    console.log("Вызван метод в экземпляре Singleton");
  }
}

// Использование Singleton

const singleton1 = Singleton.getInstance();
const singleton2 = Singleton.getInstance();

singleton1.someMethod(); // Вывод: Вызван метод в экземпляре Singleton
singleton2.someMethod(); // Вывод: Вызван метод в экземпляре Singleton

console.log(singleton1 === singleton2); // Вывод: true, так как это один и тот же экземпляр
```
